Accessibility and design

Emotion Wheel follows Tim Dixon's design system.

Design principles

  • Light, Dark, Muted Light and Muted Dark themes.
  • Roboto type with system-font fallbacks.
  • Clear navigation, card boundaries and generous spacing.
  • Visible focus indicators and controls sized for touch and keyboard use.
  • Charts paired with written summaries and data tables.
  • Colour never used as the only way information is communicated.

Keyboard and screen-reader support

  • A skip link moves directly to the main content.
  • App navigation uses a standard navigation list and normal keyboard Tab order rather than specialised menu-widget controls.
  • The current section is identified, selecting a destination moves focus to its content, and Escape closes the mobile menu and returns focus to its button.
  • Entry focus follows the visible emotion levels, reflection controls, optional note and Record action.
  • Hidden controls are removed from the visual layout and accessibility tree.
  • The mobile sticky action uses the same Record button rather than adding a duplicate control.
  • Tables include captions and column headings and become labelled row cards on small screens, avoiding sideways scrolling.
  • Charts include visible keys, written summaries and matching data tables. Shared chart images include the displayed key.

Responsive behaviour

On smaller screens, Entry, Review Logs, Review Charts, Maintenance and the theme selector collapse into one header Menu button. The selected entry actions stay reachable at the bottom of the viewport. Chart controls expose only the picker relevant to the selected day, week or month view.
